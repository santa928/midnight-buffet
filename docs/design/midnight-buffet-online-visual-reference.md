## 世界観辞書

`招待状 / 合言葉 / 控室 / 幹事 / 封蝋 / クロッシュ / 金の光 / ベルベット`

背景は既存の夜宴ホールとし、オンライン操作は「劇場の控室で招待状を確認し、幹事がクロッシュを開ける」体験として表現する。ネットワーク用語は補足文に限定し、主要 CTA には用いない。

## 採用する構造

- **Keep:** 上部の小型 HUD、中央の料理舞台、下部のベルベット製招待状パネルという三層構造。
- **Keep:** 招待コードを金縁の独立枠に置き、控室の参加者と幹事表示をその下にまとめる情報階層。
- **Keep:** 幹事の主操作を金色の大判 CTA とし、招待客には待機状態を静かに表示する。
- **Adapt:** 動的文字、コード、得点、参加者名、ボタンは画像へ焼き込まず HTML で表示する。
- **Adapt:** 実装は既存背景アセットを使い、招待状パネルを CSS で再現してロード量を増やさない。
- **Reject:** チャット操作と大きな紋章は初回オンライン版の対象外とし、操作面積と料理舞台の安全余白を優先する。

## 実装制約

- 料理舞台とオンラインパネルは上下にアンカーし、相互に重ならない。
- パネル内の CTA と席一覧は親の `padding` と `gap` から積み上げ、下端と右端がパネル境界を越えない。
- 操作ボタンは最低 `48px`、主 CTA は最低 `56px` の高さを持つ。
- 公開前の予約札の数値は本人の手札以外に描画しない。

## 検証

- `390x844` と `430x932` で、料理舞台、パネル、CTA、コード枠の重なりと横はみ出しをスクリーンショットおよび bounding box で確認する。
- 控室では幹事だけに `開宴する`、ゲーム中では幹事だけに `クロッシュを開ける` / `次の皿へ` / `もう一度乾杯` を表示する。
- 合言葉と公開前の他参加者の札が UI と RPC snapshot に含まれないことをテストする。
